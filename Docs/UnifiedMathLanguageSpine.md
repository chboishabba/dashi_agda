# Unified Math Language Spine

This repository is developing a unified mathematical language across DNA,
brain, biology, chemistry, physics, language/NLP, PNF/ITIR, braid/fabric,
Sweetgrass, dialectics, Hecke/eigen/j-invariant, BT-tree/BT-braid,
p-adic/decimal, ultrametric/adelic, wave, supervoxel, Clifford/Bott/spinor,
gauge, flow/PDE, category/functor/operator/lattice, claim/Wikidata review,
and high-n prime-carrier lanes.

The claim is not that every lane is definitionally identical to every other
lane.  The claim is that the lanes are being written in one interwoven carrier
language: observations are projected into typed carriers; carriers braid,
weave, transport, quotient, lift, restrict, remember, and compare by residual;
promotion boundaries record what has and has not been proved.

This is also the grammar for the simulator roadmap in
`Docs/SimulatorRoadmap.md`. In simulator terms, a DASHI object is not promoted
because it has evocative vocabulary. It is useful when it can be put on this
carrier grammar, projected to observables, compared by residual, and tied to a
formal or numerical receipt with explicit promotion guards.
The first executable simulator slice follows that grammar as a bounded
stellar-composition proxy diagnostic with an Agda receipt guard; it remains
non-promoting while the real EOS, opacity, reaction-network, hydrostatic, and
validation authorities are absent.

## Core Thesis

DASHI treats each domain lane as a view of structured carrier motion:

- DNA and chemistry: sequence, complement, synthesis, channel, UV, and
  supervoxel carriers.
- Brain and Brain/DNA: representation, theme, connector, codec, extraction,
  eigenbasis, and semantic-transport carriers.
- Biology: embodied sensing, animal utterance, genome/sensorimotor/connectome,
  cross-species translation, and evolutionary transport carriers.
- Physics and QFT: wave, gauge, Clifford, shell, prime-lane, moonshine,
  Yang-Mills, Navier-Stokes/flow, PDE, and closure-boundary carriers.
- Arithmetic and moonshine: Hecke scan, eigen profile, j-invariant,
  supersingular-prime, Monster, and high-n prime-carrier surfaces.
- Language/NLP and PNF/ITIR: observation, parser projection, role vector,
  predicate carrier, residual, provenance, and non-promotion surfaces.
- Claim/Wikidata review: claim atoms, relation lattices, QID/PID/value rows,
  qualifiers, references, revision windows, and non-edit-authority carriers.
- Culture/Sweetgrass/dialectics: braid, fabric, weave, knot, memory,
  reciprocity, 3/6/9, Tlurey, triadic/nonary logic, carry, and dialectical
  inclusion carriers.
- Structural mathematics: category, functor, operator, quotient, lattice,
  ultrametric, adelic, Clifford, Bott, spinor, and gauge surfaces.
- Cross-scale matter: atoms, molecules, chemistry, cells, organisms, and
  stellar composition, with strong, weak, electromagnetic, and gravitational
  carriers as stability/transition parents.

The shared grammar is:

```text
observation
  -> projection
  -> role/carrier surface
  -> thread/braid/weave composition
  -> transport or quotient/lift/restrict
  -> residual comparison
  -> review, refinement, or abstention from promotion
```

Read as a simulator workflow, the same grammar becomes:

```text
object or composition state
  -> parent lane structure
  -> unified carrier/projection surface
  -> observable or stability diagnostic
  -> numerical/formal receipt
  -> residual, review, or fail-closed non-promotion
```

The unified facade tightens this to a required object invariant:

```text
observation
  -> projection
  -> carrier/role vector
  -> residual
  -> proof posture
  -> invariant posture
  -> formal projection bundle
```

`DASHI.Unified.InvariantSpine` records this as `UnifiedObjectInvariant`.
Invariant strength and formal projection status are deliberately separate axes:

- `stronglyInvariant`: a local lane invariant is checked or definitionally
  carried by existing repo witnesses;
- `softInvariant`: the object is structurally stable as a carrier/posture
  object, but is not promoted as a theorem bridge;
- `blockedInvariant`: the object is present as an approach or boundary with
  missing proof obligations;
- `literalProjection`: the named parent projection has literal carrier
  authority in that lane;
- `softProjection`: the named parent projection is present as inherited DASHI
  carrier structure, but not promoted as an external theorem;
- `blockedProjection`: the named parent projection is present categorically, but
  the lane lacks an inhabited proof bridge.

The modular-j projection is only one member of the parent family. Every unified
object also carries Hecke, Bott, Kolmogorov, category, quotient, lattice, and
operator projections. Every formal projection in the facade has
`externalPromotion = false`.

## 369 PNF Hyperfabric

The PNF lane is now also recorded as a concrete 369 hyperfabric algebra.

The intended shape is:

```text
(n * m)^k
```

where `n` and `m` are PNF/dialectic/tri-truth encodings and `k` is the weave
depth. The canonical 369 instance is:

```text
3 * 3 = 9
9 ^ 2 = 81
3 * 3 * 3 = 27
```

The readings are:

- `3 * 3 = 9`: one nonary interaction surface;
- `9 ^ 2 = 81`: two woven dialectic interactions;
- `3 * 3 * 3 = 27`: one dialectic discussion atom / supervoxel seed;
- document time adds the fourth axis, giving a document-time hypervoxel or
  evolving PNF hyperfabric;
- p-adic and decimal valuation axes track carrier measurement;
- SSP/Monster primes label term, wheel, earth/ground, and spoke carrier roles.

This is checked in `DASHI.Interop.PNFHyperfabric369` by `refl`-level arithmetic
for the canonical dimensions and by role-vector/PNF carrier surfaces for the
document-time hyperfabric. It is an internal algebra assertion, not an external
Monster, number-theory, legal, Wikidata, or physics theorem promotion.

## Fabric Order

The repository now records a low/high order fabric distinction:

- regular fabric is low-order BT-braid-like carrier vocabulary;
- hyperfabric is high-dimensional BT-braid-like carrier vocabulary;
- high-n prime carriers, Hecke/eigen surfaces, j-invariant anchors, and
  moonshine surfaces live on the high-dimensional side.
- ultrametric/adelic, Clifford/Bott/spinor, gauge, category/functor/operator,
  and lattice/quotient surfaces are parent high-dimensional carrier surfaces.

This is a carrier classification, not a theorem that fabric, hyperfabric, and
BT-braid are definitionally equal.

## Promotion Boundary

It does promote:

- a common vocabulary for carrier comparison;
- a shared projection discipline;
- explicit bridge operations;
- residual review surfaces;

It does not automatically promote:

- legal authority;

- Hecke/eigen theorem authority;

- category equivalence authority;

- semantic truth authority;
- Navier-Stokes, Yang-Mills, or Yang-Baxter theorem claims;
- definitional equality between every domain lane.

## Asserted Proof Layer

The repository now separates carrier vocabulary from asserted internal proof
posture.

`DASHI.Interop.UnifiedMathLanguageAssertedProofs` records:

- `carrierOnly`: a lane is present as vocabulary/projection structure only;
- `internalDefinitionalProof`: local equalities are checked by computation,
  `refl`, or existing concrete witnesses;
- `internalConditionalProof`: the lane is stronger than carrier vocabulary but
  depends on named parent proof surfaces;
- `approachingClosedProof`: a lane is being treated as near closure, but not yet
  externally promoted;
- `externalPromotionBlocked`: the lane remains blocked for theorem/legal/physics
  authority outside the repo.

The asserted lane list now includes `brainFMRIObservationLane`, classified as
`carrierOnly`. It keeps the existing missing-receipt boundaries: no empirical
validation, no latent-state recovery, no cognition closure, and no qualia
closure are promoted.

This is an internal promotion from carrier/skeleton to asserted checked proof

Folder reorganization is a later phase. These parent importers are the stable
facade surface that should remain available before any child module moves are

The current status is structural, not fully quantitative. The lanes are
connected as observation transport and non-promotion surfaces, but they do not

cellular, organism, or stellar behavior.

`DASHI.Unified.CrossScaleMatterPhysics` records the intended parent ladder:

```text

  -> chemistry
  -> cells
  -> organisms
  -> stellar bodies
```

and the interaction carriers:

```text

electromagnetic

```

This is the formal place for the larger goal: composition plus force/field
carriers should eventually support stability and counterfactual questions such
as stellar composition perturbations. For now, quantitative claims like "a 3%
hydrogen perturbation makes the Sun unstable" remain explicitly blocked until
the required physical models, constants, equations of state, transport solvers,
and empirical validation receipts are inhabited.
