# Construction Applications: Four Uses of the Derivation Spine

This section should present G2, G3, G6, and E8/LILA as applications of one
derivation spine, not as a module inventory. The recurring pattern is:

1. choose a carrier with explicit coordinates;
2. define the local traversal or action visible at the current surface;
3. restrict the construction until its support, locality, or enumeration is
   witnessed;
4. ask whether the construction descends through the required quotient or
   promotion gate;
5. retain the residual obstruction as part of the mathematical object.

The four constructions below differ in domain language, but they use this same
spine. G2 applies it to local coordinate traversal, G3 to filtered
finite-support operators, G6 to distinct prime-coordinate actions, and E8/LILA
to finite local root geometry. None of these applications supplies empirical
authority, downstream physical recovery, or completed unification. Their
paper-facing value is that each one turns a possible analogy into an auditable
carrier, action, witness, and boundary.

## G2: Local Plaquettes from Coordinate Traversal

The G2 application begins with the carrier as a local coordinate chart rather
than as a field equation. A site has two visible movement directions. The
minimal local cell is therefore not an undirected point, but a two-direction
plaquette schema: move in one coordinate, move in the other, and compare the
local traversal data around the small square. This is the precise role of the
direction-indexed SFGC work. It gives the formal core a shared local schema for
current consumers that need to speak about sites, directed edges, and
plaquette-shaped neighborhoods.[^g2]

This is an application of the derivation spine because the construction starts
from traversal. The spine first asks what the carrier can see locally. Here it
can see two coordinate directions and their immediate local composition. The
positive result is that the current core-facing interfaces use this
direction-indexed local carrier consistently. That lets the paper describe a
checked local geometric interface instead of gesturing at gauge geometry in
prose.

The residual is just as important as the schema. A two-direction local
plaquette is not a Maxwell theorem, a Yang-Mills dynamics theorem, a continuum
limit, or a field-equation recovery result. It is the local coordinate surface
on which such later questions would have to be formulated. In spine terms, G2
has a carrier and local traversal schema; it has not promoted that schema into
a physical dynamics receipt.

## G3: Filtered Finite-Support Operators and the Quotient Descent Gap

The G3 application starts where local traversal becomes operator calculus. The
carrier is no longer only a site schema; it includes selected operators whose
support is finite in a witnessed sense. The current constructive result is
subtraction support for a selected finite-support operator class, not arbitrary
operator closure.[^g3-support] This matters because subtraction is one of the
first closure operations needed before a filtered operator surface can become a
stable algebraic object.

The derivation-spine reading is: restrict first, then descend. The selected
finite-support class gives a controlled prequotient surface. The next question
is whether the filtered pieces actually define an associated-graded object. For
an increasing filtration, the intended local form is:

```text
F_n / F_{n-1}
```

That notation is not enough on its own. A genuine quotient requires an accepted
equivalence modulo the denominator filtration, a quotient carrier, a proof that
projection respects the equivalence, and descended norm, product, bracket, and
isomorphism laws. Equivalently, the kernel test must be owned by the formal
surface:

```text
[x] = [0]  iff  x lies in the denominator filtration piece.
```

The current G3 boundary is therefore a quotient descent gap, not merely a list
of missing files. The spine has reached a selected finite-support prequotient
and certified one basic operation on it. It has not yet supplied the descent
machinery that would let the paper treat contraction language, norm behavior,
or brackets as quotient-correct.[^g3-gap] The admissible claim is finite-support
operator progress plus an exact quotient boundary. The inadmissible claims are
global Schrodinger recovery, arbitrary `WaveFunctionOperator` support,
witness-independent norm-law closure, product/bracket closure, or a completed
Galilei contraction theorem.

## G6: Distinct-Coordinate Commutation in Tracked FactorVec

The G6 application is the cleanest expression of coordinate independence in
the derivation spine. The carrier is a tracked `FactorVec`: a prime-coordinate
address in which different prime lanes can be updated separately. If one action
changes the p-coordinate and another action changes the q-coordinate, and if
p and q are distinct, the tracked projection should not depend on which update
is applied first. The official positive surface proves that statement for the
tracked route, with an above-threshold consumer available on the same
interface.[^g6]

This is not a loose commutativity metaphor. In spine terms, G6 identifies a
coordinate carrier, names the lane-local actions, restricts to the tracked
interface where nontrivial coordinate changes are represented, and proves the
distinct-coordinate commuting law there. The theorem is useful precisely
because it is not phrased as universal physical harmony. It is a local
compositional law for selected tracked actions on a prime-coordinate carrier.

The residual obstruction is the older universal `LaneOperator` interface. That
interface asks for a vanished-prime identity law strong enough to erase the
nontrivial scaling behavior needed by the tracked theorem. The paper should
therefore keep two sentences separate: the tracked `FactorVec` route has a
real distinct-coordinate commuting theorem; the older full interface remains
blocked and cannot be promoted by paraphrase.[^g6-block] G6 supplies tracked
coordinate compositionality, not full common-spine closure or downstream
physics closure.

## E8/LILA: Finite Local Root Geometry and Promotion Boundary

The E8/LILA application uses the same spine in an enumeration setting. The
carrier is finite local root geometry: integer and sparse local shapes can be
enumerated, audited, and compared against the local semantic boundary. The
positive paper-facing role is that a finite root-geometry surface exists for
local reasoning, including the integer/two-sparse portion and the audit that
ties the local surface to the upstream promotion question.[^e8-local]

The derivation-spine reading is again carrier, local witness, promotion gate,
and residual. The carrier is the local root-geometry enumeration. The witness
is finite and local. The promotion gate is the upstream completion receipt. The
residual is the missing promotion from local semantic finite completeness to
the upstream complete object.[^e8-gap] This distinction prevents a local
enumeration result from being restated as an upstream completion theorem.

Consequently, E8/LILA should be used in the manuscript as a finite local
root-geometry application of the spine. It may support sentences about local
enumeration, integer/two-sparse completeness, and the exact upstream-promotion
boundary. It may not support upstream `E8RootEnumerationComplete`,
E8-derived Standard Model recovery, limited SM+GR closure, GRQFT closure, or
completed unification from E8 structure.

## How the Four Applications Hang Together

These four examples are not parallel shelves in a repository. They are four
instances of the same derivation discipline:

| Application | Carrier | Local/witnessed construction | Descent or promotion question | Retained residual |
|---|---|---|---|---|
| G2 | Two-direction site coordinates | Local plaquette traversal schema | Can local geometry be promoted to dynamics? | No field-equation or continuum recovery receipt |
| G3 | Filtered selected operators | Certified finite-support subtraction | Does the prequotient descend to `F_n / F_{n-1}` with laws? | Missing quotient carrier and descended laws |
| G6 | Tracked prime-coordinate `FactorVec` | Distinct-coordinate lane actions commute | Can this replace the older universal interface? | Old `LaneOperator` promotion remains blocked |
| E8/LILA | Finite local root geometry | Integer/two-sparse local enumeration boundary | Can local semantic completeness promote upstream? | Upstream completion receipt remains missing |

This is the section's main construction claim: the formal core is organized by
a derivation spine that repeatedly converts intuition into a carrier,
witnessed local action, and explicit promotion boundary. The strength of the
paper is not that every boundary is closed. The strength is that local
geometry, operator calculus, coordinate commutation, and root enumeration are
all handled by the same fail-closed construction pattern.
